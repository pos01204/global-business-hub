# 퍼포먼스 마케터 기능 개선 기획서

## 📋 현재 문제점 진단

### 1. 크롤링 의존성 문제
- **문제**: idus 사이트의 robot.txt 제한으로 인한 크롤링 실패
- **영향**: 
  - 소재 탐색 기능이 제대로 작동하지 않음
  - 작품 정보 추출 실패 또는 불완전한 데이터
  - 샘플 데이터로 폴백하여 실제 비즈니스에 활용 불가
- **근본 원인**:
  - 웹 크롤링은 사이트 정책 변경에 취약
  - Rate limiting으로 인한 느린 응답
  - JavaScript 렌더링이 필요한 동적 콘텐츠 처리 어려움

### 2. 데이터 정확성 문제
- 크롤링 실패 시 "작품명 없음" 같은 폴백 데이터 반환
- 실제 작품 정보와 불일치
- 콘텐츠 생성 품질 저하

### 3. 워크플로우 제한
- URL 입력만으로 소재를 찾아야 함
- 수동으로 소재 정보를 입력/관리할 수 없음
- 기존 비즈니스 데이터 활용 불가

---

## 🎯 개선 목표

1. **크롤링 의존도 제거**: 수동 입력 및 데이터 연동 중심으로 전환
2. **데이터 정확성 향상**: 사용자가 직접 입력한 신뢰할 수 있는 데이터 사용
3. **워크플로우 개선**: 실제 마케팅 업무에 맞는 효율적인 프로세스 제공
4. **기존 시스템 연동**: Google Sheets 등 기존 데이터 소스 활용

---

## 🚀 개선 방안

### 방안 1: 수동 소재 관리 시스템 (우선 추천)

#### 1.1 소재 라이브러리 기능
- **기능**: 사용자가 직접 소재(작품) 정보를 입력하고 관리
- **구현**:
  - 소재 등록 폼 (제목, 설명, 이미지, 가격, 카테고리, 태그 등)
  - 소재 목록 조회 및 검색
  - 소재 수정/삭제
  - Google Sheets와 연동하여 소재 데이터 저장

#### 1.2 소재 템플릿
- 자주 사용하는 소재 유형별 템플릿 제공
- 빠른 등록을 위한 기본값 설정

#### 1.3 소재 가져오기 (URL 입력 개선)
- URL 입력 시 크롤링 시도 (선택사항)
- 크롤링 실패 시 수동 입력 폼 자동 표시
- 사용자가 정보를 직접 입력하여 등록

### 방안 2: Google Sheets 기반 소재 관리

#### 2.1 Google Sheets 연동
- 기존 Google Sheets의 작품/주문 데이터 활용
- `logistics` 시트에서 작품 정보 추출
- 자동으로 소재 라이브러리에 동기화

#### 2.2 데이터 매핑
- Google Sheets 컬럼을 소재 필드로 매핑
- 자동 변환 및 검증

### 방안 3: AI 기반 콘텐츠 생성 강화

#### 3.1 스마트 콘텐츠 생성
- 소재 정보만으로도 고품질 콘텐츠 생성
- 플랫폼별 최적화 자동 적용
- 다양한 톤앤매너 지원

#### 3.2 콘텐츠 템플릿 라이브러리
- 검증된 콘텐츠 템플릿 제공
- 카테고리별, 플랫폼별 템플릿
- 사용자 정의 템플릿 저장

### 방안 4: 통합 워크플로우

#### 4.1 소재 → 콘텐츠 → 캠페인 파이프라인
1. **소재 등록/선택**: 수동 입력 또는 Google Sheets 연동
2. **콘텐츠 생성**: AI 기반 자동 생성 또는 템플릿 사용
3. **콘텐츠 검토/수정**: 사용자가 직접 편집
4. **캠페인 생성**: 여러 콘텐츠를 묶어서 캠페인으로 관리
5. **일정 관리**: 발행 일정 설정 및 추적

#### 4.2 배치 처리
- 여러 소재를 한 번에 콘텐츠로 변환
- 일괄 편집 및 관리

## 📐 상세 기능 명세

### Phase 1: 소재 라이브러리 (우선 구현)

#### 백엔드 API
```
POST /api/marketer/materials
  - 소재 등록
  - Body: { title, description, images[], price, category, tags[], artist, metadata }

GET /api/marketer/materials
  - 소재 목록 조회
  - Query: keyword, category, limit, offset

GET /api/marketer/materials/:id
  - 소재 상세 조회

PUT /api/marketer/materials/:id
  - 소재 수정

DELETE /api/marketer/materials/:id
  - 소재 삭제

POST /api/marketer/materials/import-from-sheets
  - Google Sheets에서 소재 가져오기
```

#### 프론트엔드 UI
- **소재 등록 탭**: 
  - 수동 입력 폼
  - URL 입력 시 크롤링 시도 (선택사항)
  - 이미지 업로드
  - 카테고리/태그 선택
  
- **소재 라이브러리 탭**:
  - 소재 목록 그리드/리스트 뷰
  - 검색 및 필터링
  - 소재 선택하여 콘텐츠 생성

### Phase 2: Google Sheets 연동

#### 기능
- Google Sheets의 `logistics` 시트에서 작품 정보 추출
- 자동으로 소재 라이브러리에 등록
- 주기적 동기화 (선택사항)

### Phase 3: 콘텐츠 생성 개선

#### 기능
- 소재 정보 기반 고품질 콘텐츠 생성
- 플랫폼별 최적화 (인스타그램 해시태그, 블로그 SEO 등)
- 콘텐츠 템플릿 라이브러리
