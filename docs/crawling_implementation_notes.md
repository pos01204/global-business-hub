# idus 크롤링 구현 노트

## 구현 완료 사항

### 1. Rate Limiting
- 1분에 최대 5개 요청 제한
- 요청 간격 자동 조절
- 서버 부하 방지

### 2. 공개 정보만 수집
- 로그인 필요 정보 제외
- 작품명, 설명, 가격, 이미지 URL만 수집
- 개인정보 수집 최소화

### 3. 에러 처리
- 네트워크 오류 시 폴백 데이터 반환
- 타임아웃 설정 (10초)
- 크롤링 실패 시에도 기본 정보 제공

### 4. 데이터 제한
- 이미지 최대 10개
- 태그 최대 10개
- 제목 최대 200자
- 설명 최대 1000자

## 사용 방법

### 작품 URL로 크롤링
```typescript
const crawler = new IdusCrawler()
const product = await crawler.crawlProduct('https://www.idus.com/product/12345')
```

### 키워드 검색
```typescript
const products = await crawler.searchProducts({
  keyword: '핸드메이드',
  limit: 10
})
```

## 주의사항

1. **Rate Limiting**: 과도한 요청 방지를 위해 자동으로 요청 간격을 조절합니다.
2. **에러 처리**: 크롤링 실패 시에도 기본 정보를 반환하므로 안정적으로 작동합니다.
3. **데이터 품질**: 웹사이트 구조 변경 시 선택자를 수정해야 할 수 있습니다.
4. **법적 준수**: 공개 정보만 수집하며, 과도한 요청을 방지합니다.

## 개선 가능 사항

1. **캐싱**: 동일한 작품 정보 중복 요청 방지
2. **재시도 로직**: 일시적 오류 시 자동 재시도
3. **선택자 업데이트**: 웹사이트 구조 변경 대응
4. **이미지 다운로드**: 필요 시 이미지 로컬 저장

## 테스트 방법

1. 작품 URL 입력하여 크롤링 테스트
2. 키워드 검색 테스트
3. Rate limiting 동작 확인
4. 에러 상황 테스트 (잘못된 URL 등)
