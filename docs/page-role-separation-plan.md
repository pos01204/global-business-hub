# 페이지 역할 분리 계획: 성과 분석 vs Business Brain

**작성일**: 2024-12-11  
**목표**: 두 페이지의 명확한 역할 분리 및 상호 보완적 구조 구축

---

## 📋 현재 상태 분석

### 성과 분석 페이지 (`/analytics`)
- **현재 기능**:
  - 물류 처리 시간 분석
  - 작가별 성과 분석
  - 고객 성과 분석
  - 비교 분석 (전일/전주/전월)
- **특징**: 세부 지표 중심, 실무 작업 지향

### Business Brain 페이지 (`/business-brain`)
- **현재 기능**:
  - 경영 브리핑 (AI 기반)
  - 건강도 점수
  - 인사이트 발견
  - 고급 분석 (RFM, 코호트, 파레토, 이상 탐지)
  - 전략 제안
- **특징**: 인사이트 중심, 전략 지향

### 문제점
- 역할 모호성: 두 페이지 모두 성과 분석 기능 포함
- 중복 기능: 일부 분석이 양쪽에 존재
- 사용자 혼란: 어떤 페이지를 사용해야 할지 불명확

## 🎯 역할 분리 전략

### 성과 분석 페이지 (`/analytics`)
**역할**: **일일 운영 성과 모니터링 및 분석**

**대상 사용자**:
- 운영팀
- 물류팀
- 마케팅팀
- QC팀

**핵심 목적**:
1. **일일 운영 현황 파악**: 오늘의 핵심 지표, 긴급 이슈
2. **성과 모니터링**: 물류, 작가, 고객 성과 추적
3. **문제 발견**: 병목 지점, 지연 주문, 이상 패턴 식별
4. **작업 지향**: 즉시 조치 가능한 액션 아이템 제공

- 작업 지향적 (액션 아이템 생성)
- 간결하고 명확한 UI

### Business Brain 페이지 (`/business-brain`)
**역할**: **경영 전략 및 인사이트 제공**

**대상 사용자**:
- 경영진
- 전략팀
- 비즈니스 매니저

**핵심 목적**:
1. **전략적 의사결정**: 비즈니스 건강도, 트렌드 파악
2. **인사이트 발굴**: 통계적 인사이트, 예측, 시뮬레이션
3. **장기 계획**: 성장 기회, 위험 요소 분석
4. **AI 기반 해석**: 데이터를 비즈니스 관점에서 해석

## 🔗 페이지 간 연계 전략

### 성과 분석 → Business Brain

**연계 시나리오 1: 성과 변화 원인 분석**
```typescript

  })}

</ActionButton>
```

**연계 시나리오 2: 전략적 제안 보기**
```typescript
// 성과 분석 페이지에서 문제 발견 시

  onClick={() => navigate('/business-brain', {
    tab: 'actions',
    filter: 'logistics-bottleneck'
  })}

  전략적 제안 보기
</ActionButton>
```

## 🛠️ 구현 계획

### Phase 3.1: 역할 정의 및 구조 재설계 (2주)

**Week 1: 역할 명확화**
1. 두 페이지의 명확한 역할 정의 문서화 ✅ (이 문서)
2. 기능 분리 계획 수립
3. 사용자 워크플로우 설계

**Week 2: UI/UX 재설계**
1. 새로운 탭 구조 설계
2. 페이지 간 네비게이션 설계
3. 통합 대시보드 뷰 설계

### Phase 3.2: 성과 분석 페이지 재구성 (3주)

**Week 3-4: 기능 재구성**
1. 일일 운영 대시보드 구현
   - 오늘의 핵심 지표 카드
   - 물류 파이프라인 현황
   - 긴급 이슈 알림
   - 작가 발송 현황
2. 물류/작가/고객 성과 탭 재구성
   - 기존 기능 유지
   - Business Brain 연계 버튼 추가
3. 비교 분석 기능 강화
   - 목표 대비 달성률 추가
   - 벤치마크 비교 추가

**Week 5: 연계 기능 구현**
1. Business Brain으로의 네비게이션 추가
2. 크로스 페이지 링크 구현
3. URL 파라미터 기반 필터링

### Phase 3.3: Business Brain 페이지 재구성 (3주)

**Week 6-7: 전략 중심 재구성**
1. 경영 브리핑 탭 강화
   - EnhancedBriefingInput 활용 (v4.2)
   - 품질 검증 결과 표시
2. 인사이트 탭 재구성
   - 통계적 유의성 강조
   - 인과관계 분석 결과 표시
3. 전략 분석 탭 추가
   - 시장 분석
   - 성장 기회 발굴
   - 위험 요소 분석
4. 액션 제안 탭 추가
   - 우선순위별 액션
   - 예상 효과 시뮬레이션

**Week 8: 연계 기능 구현**
1. 성과 분석으로의 네비게이션 추가
2. 통합 대시보드 뷰 구현
3. 페이지 간 데이터 공유

### Phase 3.4: 통합 및 최적화 (1주)

**Week 9: 전체 통합 및 최적화**
1. 두 페이지 간 일관성 확보
2. 사용자 피드백 반영
3. 문서화
