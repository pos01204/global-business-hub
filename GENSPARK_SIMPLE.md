# 발표 슬라이드 생성 요청

## 발표 정보
- **제목**: 비개발자가 AI 코딩 도구로 6개월 만에 Google Sheets 대시보드 → 전용 웹 앱 3개를 구축한 이야기
- **발표자**: Global Business셀 김지훈
- **시간**: 60분
- **슬라이드**: 41장

## 디자인
- **메인 컬러**: #FF6F00 (Orange)
- **배경**: 흰색/연회색
- **강조**: Before는 빨간색, After는 녹색

---

## 슬라이드 구성

### Part 1: Opening (5장)

**1. 타이틀**
- 비개발자가 AI 코딩 도구로 6개월 만에
- Google Sheets 대시보드 → 전용 웹 앱 3개를 구축한 이야기

**2. 청중 질문**
- "매일 반복하는 업무 중 가장 귀찮은 게 뭔가요?"
- 시트 열어서 현황 확인, 여러 시트 오가며 조합...
- "저도 그랬습니다."

**3. Before/After 비교** ⭐
| 6개월 전 | 현재 |
|----------|------|
| Sheets + Apps Script | 웹 앱 3개 |
| 로딩 30-60초 | 로딩 1-2초 (30배↑) |
| 페이지 1개 | 20개+ 페이지 |
| 오프라인 매장만 | 글로벌 비즈니스 전체 |

**4. 오늘 발표 3가지**
1. 비개발자가 어떻게 웹 앱을 만들었는가
2. 데이터를 AI로 얼마나 다양하게 활용할 수 있는가
3. 여러분도 시작할 수 있다

**5. 발표자 소개**
- 김지훈 | Global Business셀
- 개발 경험 없음 (기초 HTML 수준)
- 업무: 현황 파악, 물류, 작가·작품, 고객, CS, 마케팅

---

### Part 2: 왜 자동화가 필요했는가 (6장)

**6. 업무 범위**
- 📊 비즈니스 현황 파악
- 📦 물류 대응
- 🎨 작가/작품 관리
- 👥 고객 관리
- 💬 CS 대응
- 📱 마케팅

**7. 기존 업무의 문제**
- 오전 내내 "현황 파악"에만 시간 소비
- 정작 "의사결정"은 오후부터

**8. Google Sheets 한계**
- ✅ 강점: 협업, 수식, 무료
- ❌ 한계: 시각적 표현 제한, UI/UX 커스텀 불가, 속도 저하

**9. 자동화 우선순위**
- 1순위: 반복 빈도 높음 + 시간 소요 많음

**10. Pain Point → 솔루션 매핑** ⭐
| Pain Point | 솔루션 |
|------------|--------|
| 3개 시트 오가며 KPI 계산 | 메인 대시보드 |
| 수천 행 스크롤 | 물류 관제 센터 |
| 고객 정보 조합 10분+ | 고객 360도 뷰 |
| SNS 콘텐츠 3시간 | Marketing Studio |
| 번역 외주 1-2일 | GB Translation |

**11. 핵심 깨달음**
- "데이터를 찾는 게 아니라, 판단하고 행동하는 것이 내 일"

---

### Part 3: 어떻게 만들었는가 (6장)

**12. 6개월 성장 여정** ⭐
| 6개월 전 | 현재 |
|----------|------|
| Sheets + Apps Script | 전용 웹 앱 3개 |
| 로딩 30-60초 | 로딩 1-2초 (30배↑) |
| 페이지 1개 | 20개+ (20배↑) |
| Sheets 기본 UI | 브랜드 디자인 시스템 |

**13. 핵심 비결**
- ❌ "개발 공부를 열심히 했다"
- ✅ "AI를 활용해 꾸준히 시도하고 개선했다"

**14. 개발 사이클**
Pain Point 발견 → AI에게 질문 → 프로토타입 → 테스트 → 개선 (반복)

**15. AI 자동화 가능 수준**
- Level 5: 의사결정 지원 (진행 중)
- Level 4: AI 콘텐츠 생성 ✅
- Level 3: 워크플로우 자동화 ✅
- Level 2: 데이터 가공/분석 ✅
- Level 1: 데이터 조회/표시 ✅

**16. 데이터 조합의 가치** ⭐
```
Raw Data (1차)
    ↓ AI Agent 조합
2차 데이터 (RFM 점수, 물류 상태)
    ↓
3차 데이터 (고객 세그먼트, 병목 분석)
    ↓
N차 데이터 (복합 인사이트, 추천)
```
- 핵심: "우리 데이터를 AI Agent로 얼마나 적극적/다양하게 활용하는가"

**17. 데이터 조합 실제 사례**
- 고객 360도 뷰: 3개 시트 10분 → 3초 조회

---

### Part 4: Global Business Hub (8장)

**18. 기능 맵**
| 업무 | 페이지 |
|------|--------|
| 현황 파악 | 메인 대시보드 |
| 물류 대응 | 물류 관제 센터, 미입고 관리 |
| 작품 관리 | 작품 QC |
| 고객 관리 | 고객 360도 뷰, RFM 분석 |
| CS 대응 | 통합 검색, 소포수령증 |
| 마케팅 | 쿠폰 생성기 |

**19. 메인 대시보드**
- KPI 카드 (GMV, 주문, 고객, ASP)
- 기간별 트렌드 차트

**20. 물류 관제 센터**
- 5단계 파이프라인: 입고대기→입고완료→출고준비→출고완료→배송완료
- 병목 구간 자동 하이라이트

**21. 미입고 관리**
- 작가별 미입고 현황 그룹화
- 지연일수 자동 계산

**22. 고객 360도 뷰**
- RFM 점수 & 등급
- 구매 이력, 선호 카테고리

**23. 정량적 효과**
| 업무 | Before | After | 절감 |
|------|--------|-------|------|
| 일일 현황 파악 | 1시간+ | 1분 | 98% |
| 고객 정보 조회 | 10분 | 3초 | 99% |

**24. 향후 확장 계획**
- 글로벌 작품 정보 → Notion 아카이빙 → AI Agent 연동
- 활용: 기획전 큐레이션, 작품 추천, CS 응대

**25. Hub 요약**
- 20개+ 페이지, 6개 업무 영역, 로딩 30배 개선

---

### Part 5: 추가 프로젝트 (9장)

**26-28. Marketing Studio**
- 문제: SNS 콘텐츠 작성에 3시간
- 해결: AI 콘텐츠 자동 생성 (한/일/영)
- 효과: 3시간 → 5분 (95% 절감)

**29-31. 글로벌 작가 온보딩**
- 문제: Google Forms 한계 (영상/퀴즈 불가)
- 해결: 전용 온보딩 페이지, 3단계 학습
- 효과: 브랜드 아이덴티티, 학습 완료율 향상

**32-34. GB Translation**
- 문제: 작품 번역에 30분+/건
- 해결: URL 자동 크롤링 + AI 번역 + OCR
- 효과: 30분 → 1-2분 (95% 절감)
